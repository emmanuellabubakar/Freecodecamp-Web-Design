CSS Definition

CSS (Cascading Style Sheets) specify how an html document should be displayed; the look and feel of an html document.

It allows you to apply specific styles to specific elements.

Applying CSS

Syntax for applying css:

    selector{
        property: value;
    }

There are three ways to apply css to an html documennt

  • Inline CSS: Applying styles directly into html elements using style attribute.

    Example:

        <p style="color: #808080; font-size: 18px">I am a developer</p>
  • Internal CSS: Applying styles using style element.

    Example:

        <style>
        p{
            color: #808080;
            font-size: 18px;
        }
    </style>
  • External CSS: is a seperate file for css linked to an html webpage with a css file extension placed in the head tag.

    Example:

        <link rel="stylesheet" type="text/css" href="style.css">

Selectors

It defines the element to apply css style to

  • Universal selector: selects all elements, although it can be limited to a specific namespace

    Syntax: *

  • Type selector: selects all elements that have a given node name (tags, attributes, text, comments)
  • Class selector: selects all elements with a given class attribute.
  • ID selector: selects all elements with a given id attribute.
  • Attribute selector: Selects all elements that have the given attribute.
  • MoreSelectors

    • Syntax:

        elementName{
    property: value;
    }
    .className{
    property: value;
    }
    #idName{
    property: value;
    }
    [attr] [attr=value] [attr~=value] [attr|=value] [attr^=value] [attr$=value] [attr*=value]{
    property: value;
    }
Cascade & Inheritance

Cascade

It is when styles fall from one style sheet to another, enabling multiple style sheets to be used on one HTML document.

  • The stylesheet created by the author of the page
  • The browser's default styles
  • Styles specified by the user

Inheritance

It refers to the way properties flow through the page. Unless otherwise defined, a child element will usually take on the characteristics of the parent element.

Comments

Comments are used to explain code, write notes to yourself or another developer. They are ignored by the browser.

Syntax:

<!--Some text here; it can be code, thoughts or explanation about the code-->
Styling text

  • Font-family Property

    It specifies the font name of an element and can hold several font names as a fallback system (when the browser does not support the first, it tries the next)

    There are two types of font-family property;

    • generic-name: is the name of a genric family, like serif, sans-serif, monospace, cursive, fantasy
    • family-name: is the name of a font-family, like times, helvatica, verdana

    Example:

        <p id="first-paragraph">This is the first paragraph</p>
    
    <p id="second-paragraph">This is the second paragraph</p>

        #first-paragraph{
        font-family: 'Galada', cursive;
    }
    #second-paragraph{
        font-family: monospace;
    }

    This is the first paragraph

    This is the second paragraph

    You can import custom web fonts in your website by copying the font's URL and pasting it in a link tag, in your head element before using the font in your css.

    <link href="https://fonts.googleapis.com/css?family=Verdana" rel="stylesheet" type="text/css">

    Note:

    End font-family with a generic-name as a fallback if the one stated earlier is not available.
    generic-name must not be in quotes.
    Seperate each font-family value with a comma.
    A font-name with white space must be quoted.

  • Font-size

    Specifies the size of a font. We have relative and absolute font-sizes.

    Values:

    • Keyword: xx-small, small, medium, large, larger.
    • Numericals values in length units (px, cm, pt, em, rem, vh, vw) or percentages
    • Initial
    • Inherit
    • Unset

    Note:

    When you use keywords as font-size, the user will be unable to increase the font-size.
    It is a best practice to use values that are relative to the user's default font size.

  • Font-style

    Specifies a text font style

    Values:

    • Normal: which is the default style displays a norml font style.
    • Italic: displays text in italics
    • oblique: displays an oblique font style
    • initial: specifies default value
    • inherit: Inherits font-style from its parent element

  • Font-weight

    Specifies the boldness or thickness of the text.

    Values

    • normal
    • bold
    • bolder
    • lighter
    • Number values from 100(thin) to 900(thick)

  • Font-variant

    Is used to specify if text should be displayed in a small caps font.

    Values:

    • normal
    • small-caps
    • inherit

  • Color

    Specifies the color of a text

    Values

    • Color keyword, like green, blue, red, white, transparent
    • Hexcode(#RRGGBB[AA]) which can have 6 or 3 values from 0-F
    • rgb() which is a color model that represents values as a mixture of three components; red, green and blue. Each color is between 0 to 255 e.g rgb(0, 0, 255)
    • rgba
    • hsl(): H(hue)- is an angle from 0 to 360 in deg, rad, grad or turn as unit. E.g red=0deg, grren=120deg, blue=240deg
      S(saturation) and L(lightness)- are in percentages form 0% to 100%.
    • hsla()

    Note: R- Red, G-Green, B-Blue, A-Alpha
    A(A) is for opacity and is optional with a value from 0(0%) to 1(100%)

  • Aligning text

    You can align text horizontally or vertically.

    Text-align property (horizontal align) values:

    • left
    • right
    • center
    • justify

    Aligning text vertically values:

    • top
    • middle
    • bottom
    • baseline
    • sub
    • super
    • percentages(%)
    • length(px, pt, cm)

    Note: To use vertical-align on block elements, you have to change the display of the element to table cell or inline-table

  • Text-decoration

    Specifies how the text will be decorated. It is the shorthand for; text-decoration-line text-decoration-color text-decoration-style

    Property/Values:

    • text-decoration-line: Sets the kind of text decoration to use (like underline, overline, line-through)
    • text-decoration-line: Sets the kind of text decoration to use (like underline, overline, line-through)
    • text-decoration-style Sets the style of the text decoration (like solid, wavy, dotted, dashed, double)
    • none
    • inherit
    • initial

    Note: Text-decoration-line is required but the rest is optional.

  • Indenting text

    Specifies the length of horizontal space before the beginning of a text block.

    Values:

    • length (px, pt, cm, em, etc.)
    • Percentage(%)
    • inherit

  • Text-shadow

    Adds shadow to a text, it accepts comma seperated list of shadows to a text.

    Syntax:

    text-shadow: x(offset) y(offset) blur-radius color

    Note: x offset, y offset and blur-radius have a unit in px

  • Text-transform

    Specifies how to capitalize an element's text

    Values:

    • capitalize
    • uppercase
    • lowercase

  • Letter-spacing

    Specifies the space between characters in a text

    Values:

    • normal
    • inherit
    • length(px, pt, cm, mm, etc.)

  • Word-spacing

    Specifies the space between words in a text.

    Accepts same value as letter-spacing.

  • White-spacing

    Specifies how white-space inside an element is handled

    Values:

    • normal
    • inherit
    • pre: will only wrap text on line breaks and white space
    • pre-line: will wrap text where there is a break in code, but extra white space is still ignored
    • pre-wrap: will wrap text when necessary, and on line breaks

  • Line-height

    Specifies the line height of an element

    • normal
    • number
    • length(px, pt, cm, etc.)
    • percentages(%)
    • initial
    • inherit
