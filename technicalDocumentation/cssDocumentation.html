The Basics

What is CSS

CSS (Cascading Stylls Sheets) specify how an html document should be displayed; the look and feel of an html document.

It allows you to apply specific styles to specific elements.

Applying CSS

General syntax for applying css:

    selector{
        property: value;
    }

There are three ways to apply css to an html documennt

  • Inline CSS: Applying styles directly into html elements using style attribute.

    Example:

        <p style="color: #808080; font-size: 18px">I am a developer</p>
  • Internal CSS: Applying styles using style element.

    Example:

    <> 
        <style>
        p{
            color: #808080;
            font-size: 18px;
        }
    </style>
  • External CSS: is a seperate file for css linked to an html webpage with a css file extension placed in the head tag.

    Example:

        <link rel="stylesheet" type="text/css" href="style.css">

Selectors

It defines the element to apply css style to

  • Universal selector: selects all elements, although it can be limited to a specific namespace

    Syntax: *

  • Type selector: selects all elements that have a given node name (tags, attributes, text, comments)

    Syntax: elementName

  • Class selector: selects all elements with a given class attribute.

    Syntax: .className

  • ID selector: selects all elements with a given id attribute.

    Syntax: #idName

  • Attribute selector: Selects all elements that have the given attribute.

    Syntax: [attr] [attr=value] [attr~=value] [attr|=value] [attr^=value] [attr$=value] [attr*=value]

  • More Selectors

Style Cascade & Inheritance

Cascade: is when styles fall from one style sheet to another, enabling multiple style sheets to be used on one HTML document.

  • The stylesheet created by the author of the page
  • The browser's default styles
  • Styles specified by the user

Inheritance: It refers to the way properties flow through the page. Unless otherwise defined, a child element will usually take on the characteristics of the parent element.

Comments

Comments are used to explain code, write notes to yourself or another developer. They are ignored by the browser.

Syntax:

<!--Some text here-->
